# 📋 10주차 KEYWORD 📋

## 👩🏻‍💻 APK

APK는 Android Package의 약자로, 안드로이드 앱을 기기에 직접 설치할 때 사용하는 파일 형식이다.
테스트용으로 공유하거나 사이드로딩할 때 사용할 수 있지만, 현재 Play Store 신규 앱 배포용 표준 형식은 아니다.

<br>

## 👩🏻‍💻 AAB

AAB는 Android App Bundle의 약자로, Google Play Store 배포를 위한 앱 번들 형식이다.
기기별 화면 밀도, CPU 아키텍처, 언어에 맞는 리소스만 전달할 수 있어 APK보다 앱 크기를 줄일 수 있다.

<br>

## 👩🏻‍💻 앱 서명

앱 서명은 이 앱이 개발자가 만든 정식 앱이라는 것을 증명하는 과정이다.
Play Store에 앱을 올리거나 기존 앱을 업데이트하려면 반드시 같은 서명 정보로 빌드해야 한다.

<br>

## 👩🏻‍💻 Keystore

Keystore는 Android 앱 서명에 사용되는 키 정보를 담고 있는 파일이다.
같은 앱을 업데이트하려면 최초 등록 때 사용한 Keystore가 필요하므로 절대 분실하거나 공개 저장소에 올리면 안 된다.

<br>

## 👩🏻‍💻 Release Build

Release Build는 실제 사용자에게 배포하기 위한 최종 빌드 방식이다.
디버그용 설정이 아닌 배포용 서명과 최적화가 적용된 상태로 앱을 생성한다.

<br>

## 👩🏻‍💻 bundleRelease

bundleRelease는 Gradle을 이용해 배포용 AAB 파일을 생성하는 명령어이다.
터미널에서 `./gradlew bundleRelease`를 실행하면 `app/build/outputs/bundle/release` 경로에 AAB 파일이 생성된다.

<br>

## 👩🏻‍💻 Google Play Console

Google Play Console은 Android 앱을 Play Store에 등록하고 관리하는 개발자용 플랫폼이다.
앱 기본 정보 입력, AAB 업로드, 출시 노트 작성, 심사 제출, 배포 관리 등을 이곳에서 진행한다.

<br>

## 👩🏻‍💻 스토어 등록정보

스토어 등록정보는 Play Store에서 사용자에게 보여지는 앱 소개 정보이다.
앱 이름, 짧은 설명, 자세한 설명, 스크린샷, 앱 아이콘, feature graphic 등을 작성해야 한다.

<br>

## 👩🏻‍💻 콘텐츠 등급

콘텐츠 등급은 앱의 내용이 어떤 연령대에 적합한지 분류하는 절차이다.
Google Play Console에서 설문에 답하면 앱의 기능과 콘텐츠에 따라 자동으로 등급이 부여된다.

<br>

## 👩🏻‍💻 비공개 테스트

비공개 테스트는 앱을 정식 출시하기 전에 제한된 사용자에게 먼저 배포해 검증하는 과정이다.
개인 개발자 계정은 일정 기간 동안 테스터를 대상으로 비공개 테스트를 진행한 뒤 프로덕션 출시 심사를 받을 수 있다.

<br>

## 👩🏻‍💻 Claude Code

Claude Code는 Anthropic에서 만든 터미널 기반 AI 코딩 에이전트이다.
프로젝트 전체 코드를 읽고 이해한 뒤 파일 생성, 수정, 삭제, 터미널 명령 실행, 에러 디버깅 등을 도와줄 수 있다.

<br>

## 👩🏻‍💻 CLI 기반 AI 코딩 에이전트

CLI 기반 AI 코딩 에이전트는 터미널에서 실행되며 실제 프로젝트 파일에 접근해 작업을 수행하는 AI 도구이다.
단순히 코드 조각을 답변하는 챗봇보다 프로젝트 구조를 직접 파악하고 수정할 수 있다는 점이 특징이다.

<br>

## 👩🏻‍💻 프로젝트 컨텍스트

프로젝트 컨텍스트는 AI가 작업을 이해하기 위해 필요한 프로젝트 구조, 기술 스택, 아키텍처, 코드 컨벤션 등의 정보이다.
컨텍스트를 충분히 제공하면 AI가 기존 코드 스타일과 구조에 맞는 결과물을 생성할 가능성이 높아진다.

<br>

## 👩🏻‍💻 CLAUDE.md

CLAUDE.md는 프로젝트 루트에 두는 Claude Code용 컨텍스트 문서이다.
프로젝트 개요, 아키텍처, 기술 스택, 코드 컨벤션, 주의사항 등을 작성해두면 Claude Code가 세션 시작 시 자동으로 읽고 작업에 반영한다.

<br>

## 👩🏻‍💻 하네스 엔지니어링

하네스 엔지니어링은 AI가 프로젝트에 처음 투입된 팀원처럼 일관된 방식으로 작업할 수 있도록 규칙과 컨텍스트를 문서화하는 방법이다.
CLAUDE.md를 통해 AI에게 프로젝트의 개발 규칙과 작업 기준을 주입하는 것이 핵심이다.

<br>

## 👩🏻‍💻 프롬프트 구체화

프롬프트 구체화는 AI에게 요청할 때 작업 범위, 제약조건, 기대 결과를 명확하게 작성하는 방식이다.
예를 들어 “홈 화면 만들어줘”보다 파일명, UI 구조, 사용할 라이브러리, 상태 처리 방식을 함께 알려주는 요청이 더 좋은 결과를 만든다.
